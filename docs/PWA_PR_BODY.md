# Add PWA support: manifest, service worker, icons, docs, and CI

## Summary
- Adds a Progressive Web App (PWA) implementation across the site:
  - `public/manifest.webmanifest` — web app manifest
  - `public/service-worker.js` — service worker with precache and runtime caching strategies
  - `public/sw-register.js` — client-side registration for the SW
  - `public/offline.html` — offline navigation fallback
  - `public/build/icons/` — SVG placeholders and generated PNG icons
  - `app/Http/Middleware/InjectPwaScript.php` — injects manifest & SW register tags into HTML responses
  - `tools/convert-icons.js` — Node script to create PNG icons from SVGs (uses `sharp`)
  - `package.json` scripts: `icons:generate` and `build:assets`
  - `.github/workflows/pwa-build.yml` — CI step to build assets and generate icons
  - `docs/PWA.md` — implementation notes and verification steps

## Notes & Best Practices

- Icon generation is part of the build (`npm run build:assets`) — CI runs the same script.
- Consider further SW tuning: TTLs, cache limits, and runtime cache trimming for large dynamic responses.

# Project skills

Reusable Claude Code **skills** for DMIMS live here — one folder per skill, each containing
a `SKILL.md` with the skill's frontmatter and instructions. Skills are self-contained,
repeatable workflows Claude can invoke on demand.

None are defined yet. Add one when a repeatable workflow emerges (e.g. "release a new
version", "run the full deployment dry-run", "audit tenant isolation"). See
<https://code.claude.com/docs> for the skill format.

This directory is committed so the whole team shares the same skills. Per-developer local
settings (`.claude/settings.local.json`) stay git-ignored.
